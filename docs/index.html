🐠 goldfish

The memory layer you can actually trust, built for people who run agents everywhere.

Your AI history — Claude, ChatGPT, Gemini, Claude Code, Codex, Cursor, Hermes — in one SQLite file on your machine, served to any agent over MCP. No cloud, no account, no telemetry, no Docker, no vector database.

View on GitHub Read the threat model
LocalOne SQLite file in your home directory. The only network call in the codebase is opt-in semantic search against your own Ollama.
SelectiveBound what it remembers at ingest — work chats in, personal chats never stored. Evict anything later with one command.
ScopedEach agent sees only what you grant: by source, project, git repo or time window. A hard ceiling agents can't widen.
AuditedEvery tool call logged locally — which agent searched what, how many results. Your forensic record, never your content.

Why

Your best thinking is trapped in chat windows across three apps, and every new session has a three-second memory. The memory layers that exist either ship your history to someone's cloud or want you running Postgres and a vector database to hold your own conversations.

goldfish is the other way: npm install in two seconds (zero native dependencies — built on node:sqlite), full-text search that works because you remember your own vocabulary, and an MCP server any agent can read — on terms you set.

Honest limits

Claude, ChatGPT and Gemini offer manual exports only, so you re-export every few weeks; Claude Code ingestion is fully automatic and live. Gemini's export is an activity log rather than a transcript — prompts always survive, responses when Google includes them. Connecting any memory to an agent that reads untrusted content carries prompt-injection risk — that's why scoping, redaction and the audit log exist. The threat model is documented, not hand-waved.

Get it

$ git clone https://github.com/b1rdmania/goldfish && cd goldfish
$ npm install && npm link
